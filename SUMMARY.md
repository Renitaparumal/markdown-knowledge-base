# Project Summary

## Overview
This project is a collaborative Markdown knowledge base created to demonstrate the use of Git and GitHub for version control, collaboration, and project management.

## Git & GitHub Workflow Used
The following Git features and processes were used:

- Created multiple branches (`feature-renita`, `feature-dev2`)
- Used commits with clear and descriptive messages
- Pushed changes to GitHub repository
- Created Pull Requests for each feature branch
- Reviewed and approved Pull Requests before merging
- Merged branches into the main branch
- Practiced branch-based workflow to simulate real collaboration

---

## Issues Management
- Created 2 GitHub Issues to manage project tasks
- Issues included tasks such as adding chapters and improving documentation
- All issues were marked as completed and closed after finishing work
- This helped track progress and organize project tasks effectively

---

## Challenges Faced
- Understanding how branching and merging works in Git
- Managing correct flow of Pull Requests into the main branch
- Simulating collaboration using a fake second contributor
- Ensuring proper project structure across multiple files

---

## Lessons Learned
- Git helps track changes and maintain project history
- Branching allows safe development without affecting the main codebase
- Pull Requests are useful for collaboration and code review
- GitHub Issues help manage and organize tasks effectively
- Version control is essential for teamwork and software development

---

## Conclusion
This project demonstrates practical use of Git and GitHub in a collaborative environment. It shows how version control, branching, pull requests, and issue tracking work together to support structured and efficient software development.
